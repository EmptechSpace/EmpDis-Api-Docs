## 概述

副屏控制用于在拍照流程中自动控制屏幕开关，避免屏幕反光影响拍照质量。系统通过人脸检测结果自动控制屏幕开关。

## 调用流程

在自助拍照流程中，副屏控制与拍照服务配合使用的典型流程如下：

```
1. 上层应用调用接口打开相机，设备完成初始化并返回结果
2. 上层应用发送启动（Start）指令，设备开始视频预览并进行人脸检测
3. 当设备检测到人脸后，返回"人脸已检测到"的 Trace 消息
4. 上层应用在接收到该消息后，调用关闭屏幕指令，将屏幕关闭
5. 设备在满足拍照条件后自动完成拍照，并主动返回拍照结果
6. 上层应用在获取到拍照结果后，调用开启屏幕指令，恢复屏幕显示
```

## 接口列表

### 1. 打开屏幕（Open）

当需要控制屏幕显示的时候，调用此接口，打开屏幕。

#### 请求参数

请求示例：

```json
{
  "seq": "DEV_Screen_Open_${uuid}",
  "cmd": "Open",
  "datetime": "20211201130101",
  "posidx": "00",
  "timeout": "30000",
  "async": "0"
}
```

参数说明：

| 参数名称 | 格式 | 是否必填 | 参数说明 |
|----------|------|----------|----------|
| seq | string | 是 | DEV_Screen_Open_${uuid} |
| cmd | string | 是 | 固定为"Open" |
| datetime | string | 是 | 指令的下发时间，格式：YYYYMMddHHmmss |
| posidx | string | 是 | 多个同款设备的工位号；"00"~"99" |
| timeout | string | 是 | 超时时间(ms) |
| async | string | 是 | 是否异步（默认0:同步）；0：同步；1：异步 |

#### 返回参数

返回示例：

```json
{
  "seq": "DEV_Screen_Open_${uuid}",
  "cmd": "Open",
  "datetime": "20211201130102",
  "code": "0",
  "msg": "Success",
  "suggest": "",
  "posidx": "00"
}
```

参数说明：

| 参数名称 | 格式 | 是否必填 | 参数说明 |
|----------|------|----------|----------|
| seq | string | 是 | 同下发的 seq |
| cmd | string | 是 | 同下发的 cmd |
| datetime | string | 是 | 指令的下发时间，格式：YYYYMMddHHmmss |
| code | string | 是 | 参照通用返回码 / 副屏控制返回码 |
| msg | string | 否 | 提示信息 |
| suggest | string | 否 | 建议 |
| posidx | string | 是 | 多个同款设备的工位号；"00"~"99" |
