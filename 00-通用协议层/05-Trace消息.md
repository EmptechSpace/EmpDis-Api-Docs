# 外设 Trace 消息

## 文档版本

| 版本 | 日期 | 修改内容 |
|------|------|----------|
| V1.0 | 2026-06-16 | 初始版本，从原始文档拆分 |

## 概述

部分外设的指令，执行过程中会有一些交互的 Trace 消息（如拍照服务、指纹仪）。上层应用不需要发送请求指令，只是正在执行的指令的外设主动返回的消息。返回的 Trace 消息的 seq、cmd 和当前正在执行的指令的 seq、cmd 相同，Trace 消息的 code 值固定为"1031"。

上层应用在收到此类消息时，可以根据自身的需求，选择性对 Trace 消息进行指定操作。比如：在收到指纹仪提醒抬起手指的 Trace 消息后，上层应用可以自主播放抬起手指的语音对用户进行提醒。

## 调用流程

1. 上层应用给外设发送了一个请求指令（不固定）
2. 外设收到请求指令，开始执行
3. 执行过程中连续的多次发送 Trace 消息
4. 外设执行完指令后，返回操作结果给上层应用

## 返回参数

返回示例：

```json
{
  "seq": "DEV_FPrint_FgVerify_${uuid}",
  "cmd": "FgVerify",
  "datetime": "20211201130102",
  "code": "1031",
  "msg": "trace message",
  "data": {
    "event_type": "2",
    "tip_code": "9",
    "tips": "Please place finger"
  }
}
```

参数说明：

| 参数名称 | 格式 | 是否必填 | 参数要求 |
|----------|------|----------|----------|
| seq | string | 是 | 同正在执行的指令的 seq |
| cmd | string | 是 | 同正在执行的指令的 cmd |
| datetime | string | 是 | 指令的返回时间，格式：YYYYMMddHHmmss |
| code | string | 是 | 固定值："1031" |
| msg | string | 是 | trace message |
| data | object | 是 | 不同外设返回的不同，参考对应外设的说明 |
