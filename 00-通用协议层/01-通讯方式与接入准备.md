## DIS服务介绍

传统自助设备的上层应用，通常需要直接对接摄像头、指纹仪、身份证阅读器等各类外设硬件。面对外设品牌多、协议繁杂的现状，上层应用需投入大量研发资源适配不同设备指令；同时，随着硬件迭代加速，每当新增或更换外设型号，应用层都需重新开发对接接口，导致研发成本重复投入、接口复用率低、整体效率难以提升。

针对上述痛点，雄帝科技依托多年设备领域技术沉淀，推出 DIS 外设服务。由 DIS 服务统一封装各类外设的协议解析与功能指令，向下完成全品类硬件的标准化对接；上层应用仅需对接 DIS 服务，即可实现对所有外设的统一调用，真正做到一类设备仅需一次对接。

通过该模式，可大幅减少上层应用在硬件适配环节的重复研发投入，降低系统耦合度，让业务开发更聚焦、迭代更高效，实现真正的提效减负。

## 整体架构

该架构采用分层设计：上层业务系统通过统一接口与 EmpDIS 交互，EmpDIS 作为设备接口服务中枢，负责屏蔽底层设备差异、统一指令（如 Open/Read/TakePhoto/Print 等）、并管理设备状态、异常处理和重试机制；底层则连接各类外设，包括摄像头、护照阅读器、打印机、扫码设备以及控制板等，从而实现上层与多设备之间的解耦与统一控制。

## 通讯方式

默认提供 WebSocket 通讯方式，特殊情况下，也可提供 HTTP 1.x 的协议接口。

## 通讯地址

- WebSocket：`ws://127.0.0.1:16001/dis/perso`
- HTTP：`http://127.0.0.1:16001/dis/perso`

采用 WebSocket 通讯方式时，在调用管理接口或者外设类接口时需要先与 EmpDIS 建立通讯连接。

## 通讯配置

WebSocket 参数配置说明：

| 参数 | 说明 |
|------|------|
| 通讯超时时间 | 可按照需求设定，并无强制要求。采用 WebSocket 通讯方式时，超时时间通常默认为 30000~50000ms |

## 报文格式

| 项目 | 说明 |
|------|------|
| 服务端 | 外设接口服务 EmpDIS |
| 客户端 | 上层 APP 程序 |
| 报文格式 | 数据以 JSON 方式组报文 |
| 链接方式 | TCP 长连接（WebSocket） |
| 交互方式 | 1问1答（WebSocket、HTTP）；亦可1问多答（WebSocket） |
| 校验方式 | MD5 |
| 最大长度 | 暂定单包最大 1MB |
| 系统要求 | Windows7 专业版及其以上 |
| 硬件要求 | CPU: [x86_32、x86_64、AMD64、ARM]；RAM:[8GB+]；HDD:[500GB+] |
