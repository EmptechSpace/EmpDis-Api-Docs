# 获取设备视频流

## 文档版本

| 版本 | 日期 | 修改内容 |
|------|------|----------|
| V1.0 | 2026-06-16 | 初始版本，从原始文档拆分 |

## 概述

本指令用于获取具备视频流交互能力的设备（如摄像头、护照阅读器、虹膜仪、指纹仪）的实时视频流数据。调用后设备将返回视频流地址或持续推送图像数据，供上层应用进行显示或处理。

## 调用流程

1. **打开设备（Open）**
   - 请求：Open
   - 返回：WebSocket 地址
   - 说明：设备初始化视频流服务，返回一个用于视频传输的 WebSocket 地址，例如：`ws://127.0.0.1:26034/dis/hi_video`

2. **获取 WebSocket 地址**
   - 上层应用接收设备返回的 WebSocket 地址
   - 说明：该地址用于后续建立视频流连接，此阶段不会返回视频数据

3. **建立 WebSocket 连接**
   - 上层应用通过获取到的地址建立 WebSocket 连接
   - 连接地址：`ws://127.0.0.1:26034/dis/hi_video`
   - 说明：与设备建立长连接，为后续视频数据传输做准备

4. **发送取流指令（TakeVideo）**
   - WebSocket 连接建立后，上层应用发送 TakeVideo 指令
   - 说明：通知设备开始推送视频流，进入持续视频传输状态

5. **接收视频流数据**
   - 设备持续向上层应用推送视频帧数据
   - 数据格式：Base64 图片数据
   - 说明：每一帧为一张 Base64 编码的图片，持续推送，形成视频流效果

6. **视频显示**
   - 上层应用对接收到的 Base64 数据进行解析并展示
   - 说明：将 Base64 解码为图片，按帧实时渲染到界面上

## TakeVideo 请求参数

请求示例：

```json
{
  "seq": "DEV_Camera_TakeVideo_${uuid}",
  "cmd": "TakeVideo",
  "datetime": "20211201130101",
  "timeout": "30000"
}
```

参数说明：

| 参数名称 | 格式 | 是否必填 | 参数要求 |
|----------|------|----------|----------|
| seq | string | 是 | DEV_Camera_TakeVideo_${uuid}，uuid 作为指令的唯一号 |
| cmd | string | 是 | 固定为"TakeVideo" |
| datetime | string | 是 | 指令的下发时间，格式：YYYYMMddHHmmss |
| timeout | string | 是 | 超时时间(ms) |

## TakeVideo 返回参数

返回示例：

```json
{
  "seq": "DEV_Camera_TakeVideo_${uuid}",
  "cmd": "TakeVideo",
  "datetime": "20211201130102",
  "code": "0",
  "msg": "video stream",
  "VideoFrame": "pic_base64"
}
```

参数说明：

| 参数名称 | 格式 | 是否必填 | 参数要求 |
|----------|------|----------|----------|
| seq | string | 是 | 同下发的 seq |
| cmd | string | 是 | 同下发的 cmd |
| datetime | string | 是 | 指令的返回时间，格式：YYYYMMddHHmmss |
| code | string | 是 | 参照通用返回码 |
| msg | string | 是 | video stream |
| VideoFrame | string | 是 | 图片的 Base64 数据 |
