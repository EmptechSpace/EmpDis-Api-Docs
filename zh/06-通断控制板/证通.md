## 调用流程

当硬件正常使用过程中出现打开设备失败的情况下，可考虑对其进行复位，并重新打开硬件。

```
OnOffBoard（通断板通断）
Reset（通断板复位）
```

## 接口列表

### 1. 通断板通断（OnOffBoard）

控制通断板的指定通道进行通断。

#### 请求参数

请求示例：

```json
{
  "seq": "DEV_IOControl_OnOffBoard_${uuid}",
  "Timeout": "30000",
  "cmd": "OnOffBoard",
  "datetime": "20211201130101",
  "param": {
    "Channel": "4",
    "Nid": "1",
    "Status": "1"
  },
  "posidx": "00",
  "async": "0"
}
```

参数说明：

| 参数名称 | 格式 | 是否必填 | 参数说明 |
|----------|------|----------|----------|
| seq | string | 是 | 请求序列号：字段格式为业务标识头+下划线+唯一号 |
| cmd | string | 是 | 本指令下，固定为"OnOffBoard" |
| datetime | string | 是 | 指令的下发时间，格式：YYYYMMddHHmmss |
| posidx | string | 是 | 多个同款设备的工位号；"00"~"99" |
| Timeout | string | 是 | 超时时间(ms) |
| async | string | 是 | 是否异步（默认0:同步）；0：同步；1：异步 |
| param | object | 是 | 请求参数 |
| ↳ Nid | string | 是 | 通断板序号，值为1到4（支持连接多个通断板，一般只接一个通断板的时候就填1） |
| ↳ Channel | string | 是 | 通断板上 USB 通道号，值为1到4（对应通断板上1到4的 USB 接口） |
| ↳ Status | string | 是 | 对 Channel 通道号的通断操作：1 连通、2 断开、3 复位 |

#### 返回参数

返回示例：

```json
{
  "seq": "DEV_IOControl_OnOffBoard_${uuid}",
  "Timeout": "30000",
  "cmd": "OnOffBoard",
  "datetime": "20211201130101",
  "data": {},
  "code": "0",
  "posidx": "00",
  "async": "0"
}
```

参数说明：

| 参数名称 | 格式 | 是否必填 | 参数说明 |
|----------|------|----------|----------|
| seq | string | 是 | 同下发的 seq |
| cmd | string | 是 | 同下发的 cmd |
| datetime | string | 是 | 指令的下发时间，格式：YYYYMMddHHmmss |
| code | string | 是 | 参照通断控制板错误码说明 |
| msg | string | 否 | 参照通断控制板错误码说明 |
| posidx | string | 是 | 多个同款设备的工位号；"00"~"99" |

## 错误码

| 序号 | 错误码 | 含义 |
|------|--------|------|
| 1 | 17690001 | 初始化失败 |
| 2 | 17690002 | 通断失败 |
| 3 | 17690003 | 复位失败 |
| 4 | 17690004 | 获取库版本失败 |
| 5 | 17696001 | 命令不支持 |

> 通用返回码（0~1037）请参阅 [通用返回码](../00-通用协议层/06-通用返回码.md)
